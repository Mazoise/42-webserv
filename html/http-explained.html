Webserviette

What is HTTP ?

HTTP est un protocole de communication client-serveur. Les clients HTTP les plus connus sont les navigateurs Web permettant à un utilisateur d'accéder à un serveur. Il existe des clients en ligne de commande comme curl ou telnet par exemple. Le client envoie un requête au serveur, et celui-ci renvoie une reponse en fonction des ressources en local et des éléments de la requête.

Pro-tips : Vous pouvez observez les reponses serveur et leurs requêtes associés à l'aide de l'outils de développement Ctrl Maj I onglet Réseaux.

Requête et methodes

La requête est un message composé de plusieurs lignes. La première ligne contient La methode et la cible de la methode.

GET /index.html HTML/1.1

Les méthodes gerés par ce serveur sont les suivantes :

  • GET : Acceder à la ressource
  • DELETE : Supprimer la ressource
  • POST : Envoyer des données via le body

La suite de la requête est composé de Header, chaque header est composé d'une clé associé à une valeur. Ces valeurs peuvent être utilisées par le serveur.

. . .
Host: localhost:8081
Accept-Encoding: gzip, deflate
Accept-Language: fr_fr
Connection: keep-alive
. . .

Enfin la requête peut se terminer par un body contenent des informations, comme des données de formulaire. Ces données peuvent étre utilisé par un programme tier par exemple.

Code et Reponse

Le serveur va pouvoir executer des actions et renvoyer une reponse. Chaque reponse contient un code indiquant si tout c'est bien passé ou si une erreur est survenue.

  • Succès de la requête
    HTTP/1.1 200 OK
  • Redirection
    HTTP/1.1 302 Found
  • Ressource introuvable (Erreur requête)
    HTTP/1.1 404 Not Found
  • Erreur serveur
    HTTP/1.1 500 Internal Server Error

Comme pour la requête, la reponse contient des headers qui seront utiliser par le client. Un body peut etre revoyer. Par exemple pour une methode GET, le body contiendra la ressource demander.

HTTP/1.1 200 OK
Content-Length: 1643
Content-type: image/png
Server: Webserviette/0.1

[BODY]

Et les CGI ?

Au lieu d'envoyer le contenu d'un fichier (fichier HTML, image), le serveur HTTP exécute un programme, puis retourne le contenu généré. CGI est le standard industriel qui indique comment transmettre la requête du serveur HTTP au programme, et comment récupérer la réponse générée dudit programme.
